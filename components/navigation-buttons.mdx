## Overview

The navigation button component provides left and right controls for the carousel with a subtle pulsing animation effect. Buttons are conditionally displayed based on the current selection position and feature a dark circular background with white icons.

## Implementation

### _buildNavigationButton()

```dart
Widget _buildNavigationButton({
  required IconData icon,
  required VoidCallback onPressed,
  required bool isLeft,
}) {
  return AnimatedBuilder(
    animation: _buttonAnimationController,
    builder: (context, child) {
      return Transform.scale(
        scale: _buttonAnimation.value,
        child: Container(
          decoration: const BoxDecoration(
            color: Colors.black54,
            shape: BoxShape.circle,
          ),
          child: IconButton(
            icon: Icon(icon, color: Colors.white),
            onPressed: onPressed,
            iconSize: 30,
          ),
        ),
      );
    },
  );
}
```

## Animation System

### Animation Controller

The pulse animation is controlled by a dedicated `AnimationController`:

```dart
late AnimationController _buttonAnimationController;
late Animation<double> _buttonAnimation;
```

Source: `lib/main.dart:67-68`

### Animation Initialization

```dart
void _initializeButtonAnimation() {
  _buttonAnimationController = AnimationController(
    vsync: this,
    duration: _buttonPulseDuration,
  )..repeat(reverse: true);

  _buttonAnimation = Tween<double>(begin: 1.0, end: 1.3).animate(
    CurvedAnimation(
      parent: _buttonAnimationController,
      curve: Curves.easeInOut,
    ),
  );
}
```

Source: `lib/main.dart:88-100`

### Animation Properties

<ResponseField name="duration" type="Duration" default="1500ms">
  Full cycle duration for the pulse animation.
  
  Defined by `_buttonPulseDuration` constant.
</ResponseField>

<ResponseField name="scaleRange" type="Tween<double>">
  Scale transformation range from 1.0 to 1.3.
  
  - **Minimum**: 1.0 (normal size)
  - **Maximum**: 1.3 (30% larger)
</ResponseField>

<ResponseField name="curve" type="Curve" default="Curves.easeInOut">
  Smooth acceleration and deceleration for natural pulsing effect.
</ResponseField>

<ResponseField name="repeat" type="bool" default="true">
  Animation repeats infinitely with reverse mode.
  
  Creates continuous breathing effect: 1.0 → 1.3 → 1.0 → 1.3 ...
</ResponseField>

## AnimatedBuilder Pattern

The button uses `AnimatedBuilder` for efficient animation:

```dart
AnimatedBuilder(
  animation: _buttonAnimationController,
  builder: (context, child) {
    return Transform.scale(
      scale: _buttonAnimation.value,
      child: Container(
        // ... button content
      ),
    );
  },
)
```

<Note>
  `AnimatedBuilder` only rebuilds the necessary parts of the widget tree, improving performance over rebuilding the entire widget.
</Note>

## Conditional Rendering

Buttons are conditionally displayed based on carousel position:

### Left Navigation Button

```dart
if (_selectedIndex > 0)
  Positioned(
    left: 20,
    child: _buildNavigationButton(
      icon: Icons.arrow_back_ios,
      onPressed: _selectPreviousItem,
      isLeft: true,
    ),
  ),
```

Source: `lib/main.dart:223-231`

**Display Condition**: Only shown when `_selectedIndex > 0` (not at first item)

**Position**: 20 pixels from left edge

### Right Navigation Button

```dart
if (_selectedIndex < _items.length - 1)
  Positioned(
    right: 20,
    child: _buildNavigationButton(
      icon: Icons.arrow_forward_ios,
      onPressed: _selectNextItem,
      isLeft: false,
    ),
  ),
```

Source: `lib/main.dart:234-242`

**Display Condition**: Only shown when `_selectedIndex < _items.length - 1` (not at last item)

**Position**: 20 pixels from right edge

<Tip>
  Hiding buttons at the list boundaries prevents users from attempting to navigate beyond available items.
</Tip>

## Navigation Functions

### _selectPreviousItem()

```dart
void _selectPreviousItem() {
  if (_selectedIndex > 0) {
    _selectItem(_selectedIndex - 1);
  }
}
```

Source: `lib/main.dart:116-120`

Decrements the selected index with boundary checking.

### _selectNextItem()

```dart
void _selectNextItem() {
  if (_selectedIndex < _items.length - 1) {
    _selectItem(_selectedIndex + 1);
  }
}
```

Source: `lib/main.dart:123-127`

Increments the selected index with boundary checking.

<Note>
  Both functions include boundary checks even though the buttons are conditionally rendered, providing defensive programming.
</Note>

## Advanced Features

<Tabs>
  <Tab title="Continuous Animation">
    The pulse animation runs continuously while buttons are visible:
    
    - Starts in `initState()`
    - Runs until widget disposal
    - Independent of carousel interactions
    - No performance impact due to efficient `AnimatedBuilder`
    
    This creates an always-present visual cue for navigation.
  </Tab>
  
  <Tab title="Transform.scale">
    Uses `Transform.scale` for GPU-accelerated animation:
    
    - Scales from center point
    - Does not affect layout of surrounding widgets
    - Smooth performance on all devices
    - Preserves hit test area
  </Tab>
  
  <Tab title="State Synchronization">
    Button visibility syncs with carousel state:
    
    - Updates on every `setState()` call
    - Conditional rendering in build method
    - No additional state tracking needed
    - Automatic hide/show on navigation
  </Tab>
</Tabs>

## Integration with Carousel

Buttons are positioned within the same `Stack` as carousel items:

```dart
Stack(
  alignment: Alignment.center,
  children: [
    // Carousel items
    ...List.generate(
      _items.length,
      (index) => _buildCarouselItem(context, index),
    ),
    
    // Navigation buttons overlay
    if (_selectedIndex > 0)
      Positioned(
        left: 20,
        child: _buildNavigationButton(...),
      ),
    if (_selectedIndex < _items.length - 1)
      Positioned(
        right: 20,
        child: _buildNavigationButton(...),
      ),
  ],
)
```

<Tip>
  Buttons are rendered after carousel items in the `Stack`, ensuring they appear on top and remain clickable.
</Tip>
